## Codebase Patterns
- Server list endpoints use dedicated store classes under `server/src/models`, and route handlers under `server/src/routes` are kept thin.
- Non-production list endpoints can fall back to fixture data when Prisma is unavailable or a table is missing (`P2021`), which keeps tests and local work unblocked.
- Backend tests use Jest + Supertest with `../lib/prisma` mocked directly inside each route/store test file.
- Route registration for API modules is centralized in `server/src/routes/index.ts`.
- The Expo Router mobile app keeps tab registration in `mobile/app/_layout.tsx`, while `mobile/app/index.tsx` is the home shortcut hub for top-level modules.
- Mobile validation currently uses `cd mobile && npm test` for lightweight source-oriented Jest checks and `cd mobile && npm run typecheck` for TypeScript verification.
- When Expo tabs use `@expo/vector-icons`, preload the icon font in `mobile/app/_layout.tsx` with `useFonts(Ionicons.font)` to avoid missing/garbled tab icons during web rendering.
- The current mobile regression tests are source-level Jest assertions under `mobile/app/__tests__`, which makes top-level navigation semantics cheap to verify without full native rendering.
